Platform Overview

What is Agentic Platform?

Agentic Platform is a comprehensive, containerized data processing platform designed for intelligent data ingestion, transformation, storage, and output across multiple formats and destinations.

Core Architecture

Input Layer
Data ingestion from multiple sources
Agentic Brain
Intelligent processing & vector operations
Output Layer
Multi-format data delivery

Key Capabilities

  • Multi-Source Ingestion: CSV, Excel, PDF, JSON, APIs, Web Scraping
  • Semantic Search: AI-powered vector similarity search
  • Data Transformation: Intelligent format conversion and validation
  • Enterprise Security: OAuth2/OIDC, TLS, field-level encryption
  • Monitoring: Prometheus, Grafana, Jaeger distributed tracing
Quick Start
  1. Configure environment variables
  2. Run docker-compose up
  3. Access UI at http://localhost:8082
  4. Upload data for processing
  5. Monitor via Grafana dashboard
System Requirements
  • Docker & Docker Compose
  • 8GB+ RAM recommended
  • 4+ CPU cores
  • 50GB+ disk space
  • Linux/Windows/Mac OS

Troubleshooting Guide

Common Issues & Solutions

Problem:

Docker containers fail to start due to port conflicts.

Solution:
  1. Check which ports are in use: lsof -i :PORT
  2. Use port manager service: docker-compose up port-manager
  3. Update .env file with suggested ports
  4. Restart services: docker-compose up -d
Prevention:
  • Always use the port manager before starting services
  • Configure unique port ranges for different environments
  • Use docker-compose.override.yml for local development

PostgreSQL Connection Failed:
# Check if PostgreSQL is running docker-compose ps postgresql_ingestion # View logs docker-compose logs postgresql_ingestion # Test connection docker-compose exec postgresql_ingestion psql -U agentic_user -d agentic_ingestion
MongoDB Connection Issues:
# Check MongoDB status docker-compose ps mongodb_output # Test connection docker-compose exec mongodb_output mongo -u agentic_user -p
Qdrant Connection Issues:
# Check Qdrant status curl http://localhost:6333/health # View collections curl http://localhost:6333/collections

Check Service Status:
# View all services docker-compose ps # Check specific service logs docker-compose logs ingestion-coordinator # Test health endpoints curl http://localhost:8080/health curl http://localhost:8081/health curl http://localhost:8082/health
Common Health Issues:
  • Starting: Service is initializing, wait 30-60 seconds
  • Unhealthy: Check logs for error messages
  • Exited: Service crashed, check logs for stack traces

Memory Issues:
# Check container resource usage docker stats # View specific container logs docker-compose logs ingestion-coordinator | grep -i memory # Adjust Docker memory limits docker-compose.yml: services: ingestion-coordinator: deploy: resources: limits: memory: 2G reservations: memory: 1G
Slow Processing:
  • Increase BATCH_SIZE for better throughput
  • Add more workers: MAX_WORKERS=8
  • Optimize database indexes
  • Use Redis caching for frequently accessed data
Vector Search Performance:
  • Reduce SEARCH_LIMIT for faster queries
  • Use appropriate SCORE_THRESHOLD
  • Consider smaller embedding models for speed
  • Implement result caching

REQUIRE_AUTH=true Issues:
# Check JWT configuration echo $JWT_SECRET echo $JWT_EXPIRATION_HOURS # Test OAuth2 configuration echo $OAUTH_CLIENT_ID echo $OAUTH_CLIENT_SECRET echo $OIDC_ISSUER_URL
API Authentication:
# Include Authorization header curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \ http://localhost:8080/protected-endpoint # For OAuth2, obtain token first curl -X POST https://your-oauth-provider/token \ -d "grant_type=password&username=USER&password=PASS&client_id=CLIENT_ID"
Disable Authentication for Testing:
# Set in .env file REQUIRE_AUTH=false # Restart services docker-compose restart
