Platform Overview

What is Agentic Platform?

Agentic Platform is a comprehensive, containerized data processing platform designed for intelligent data ingestion, transformation, storage, and output across multiple formats and destinations. Built for enterprises that need scalable, secure, and AI-powered data processing capabilities.

🏗️ Three-Tier Architecture

Input Layer

Multi-source data ingestion with intelligent processing

Agentic Brain

AI-powered processing, vector operations, and orchestration

Output Layer

Multi-format data output with routing intelligence

Key Capabilities
  • Multi-source data ingestion
  • AI-powered vector operations
  • Enterprise-grade security
  • Real-time monitoring
Supported Technologies
  • PostgreSQL, MongoDB, Qdrant
  • FastEmbed, OpenAI integration
  • 100% Docker containerized
  • Prometheus, Grafana monitoring

Prerequisites & System Requirements
Hardware Requirements
  • RAM: Minimum 8GB, Recommended 16GB+
  • CPU: 4+ cores with AVX2 support
  • Storage: 20GB+ free disk space
  • Network: Stable internet connection
Software Requirements
  • Docker: Version 20.10+ with Docker Compose
  • Operating System: Linux, macOS, or Windows with WSL2
  • Git: For cloning the repository
  • Browser: Modern browser (Chrome, Firefox, Safari, Edge)
Verification Commands: 
# Check Docker installation
docker --version
docker-compose --version

# Check available resources (Linux/macOS)
free -h || echo "RAM check"
df -h | grep -E "(Filesystem|/$)" || echo "Disk space check"

# Check AVX2 support (Linux)
grep avx2 /proc/cpuinfo || echo "AVX2 not detected"
Installation & Setup
1 Clone Repository
2 Configure Environment
3 Start Services
4 Verify Installation
Step 1: Clone the Repository
# Clone the Agentic Platform repository
git clone https://github.com/your-org/agentic-platform.git
cd agentic-platform

# Verify the contents
ls -la

Expected output: You should see docker-compose.yml, schema.sql, and various service directories.

Step 2: Configure Environment Variables
Important: Review and customize the environment variables before starting services. 
# Copy the example environment file
cp .env.example .env

# Edit the configuration (use your preferred editor)
nano .env  # or vim .env or code .env
Key Configuration Options:
Variable Description Default Required
REQUIRE_AUTH Enable/disable authentication false Optional
JWT_SECRET Secret key for JWT tokens Auto-generated If AUTH enabled
POSTGRES_PASSWORD Database password Auto-generated Auto
QDRANT_API_KEY Vector database API key Auto-generated Auto
VECTOR_UI_PORT Vector UI service port 8082 Optional
Step 3: Start All Services
First startup may take 5-10 minutes as Docker images are built and services initialize. 
# Make the startup script executable
chmod +x start-platform.sh

# Start all services
./start-platform.sh

# Alternative: Start with Docker Compose directly
docker-compose up -d --build

What happens during startup:

  • Docker images are built for all services
  • Databases (PostgreSQL, Qdrant, Redis) are initialized
  • Message queues (RabbitMQ) are configured
  • Vector embeddings models are downloaded
  • Services perform health checks and connect to each other
Step 4: Verify Installation
Check Service Status:
# Check all services are running
docker-compose ps

# View service logs
docker-compose logs --tail=50

Troubleshooting

Common Issues & Solutions

Resolve common platform issues quickly

Service Won't Start

Check if ports are available:

netstat -tulpn | grep :8080

Stop conflicting services or change ports in .env

Memory Issues

Increase Docker memory limit to 8GB+ in Docker Desktop settings

Connection Refused

Wait for PostgreSQL to fully start (may take 30-60 seconds)

docker-compose logs postgres
