Troubleshooting Guide - Common Issues & Solutions

Quick Diagnosis: Use the navigation below to jump to specific issues

üöÄ Startup & Initialization Issues

Problem: Docker is not running
# Check Docker status docker info # Start Docker (macOS) # Open Docker Desktop # Start Docker (Linux) sudo systemctl start docker
Problem: Port conflicts
# Check what's using ports lsof -i :8080 lsof -i :8081 lsof -i :8082 # Kill process using port sudo kill -9
Problem: Insufficient resources
# Check available memory free -h # Check Docker resource limits docker system df # Clean up Docker docker system prune -a

Problem: Services fail to start
# Check service logs docker-compose logs ingestion-coordinator docker-compose logs output-coordinator # Restart specific service docker-compose restart ingestion-coordinator # Check service health curl http://localhost:8080/health
Problem: Database connection failures
# Check database logs docker-compose logs postgresql_ingestion # Test database connection docker-compose exec postgresql_ingestion psql -U agentic_user -d agentic_ingestion # Reset database docker-compose down -v docker-compose up -d

Problem: Embedding generation fails
# Check Qdrant status curl http://localhost:6333/health # Test embedding endpoint curl -X POST http://localhost:8082/api/embeddings/generate \ -H "Content-Type: application/json" \ -d '{"texts": ["test text"]}' # Check logs docker-compose logs vector-ui
Problem: Vector search returns no results
# Check collection exists curl http://localhost:6333/collections # Index test document curl -X POST http://localhost:8081/vectors/index \ -H "Content-Type: application/json" \ -d '{"documents": [{"document_id": "test", "text": "test content"}]}' # Search curl -X POST http://localhost:8081/vectors/search \ -H "Content-Type: application/json" \ -d '{"query": "test", "collection": "documents"}'

Problem: Slow processing
# Check resource usage docker stats # View Prometheus metrics curl http://localhost:9090/api/v1/query?query=up # Adjust batch size export BATCH_SIZE=500 docker-compose restart ingestion-coordinator
Problem: Memory issues
{% raw %} # Check memory usage docker stats --format "table {{.Container}}\t{{.CPUMem}}" # Increase memory limits # Edit docker-compose.yml # Add memory limits to services {% endraw %}
Problem: High CPU usage
{% raw %} # Check CPU usage docker stats --format "table {{.Container}}\t{{.CPUPerc}}" # Adjust worker count export MAX_WORKERS=2 docker-compose restart {% endraw %}

Problem: Service communication failures
# Check network connectivity docker network ls docker network inspect agenticai_agentic-network # Test service connectivity docker-compose exec ingestion-coordinator curl output-coordinator:8081/health # Restart network docker-compose down docker network prune -f docker-compose up -d
Problem: External API failures
# Check internet connectivity curl https://httpbin.org/get # Test DNS resolution nslookup google.com # Check proxy settings env | grep -i proxy

Problem: File processing failures
# Check file permissions ls -la /path/to/files # Validate file format head -10 file.csv file file.pdf # Check service logs docker-compose logs csv-ingestion
Problem: Data validation errors
# Check validation rules curl http://localhost:8080/validation/rules # Test data validation curl -X POST http://localhost:8080/validate \ -H "Content-Type: application/json" \ -d '{"data": "test"}' # View error logs docker-compose logs | grep -i validation
Problem: Duplicate data issues
# Check deduplication settings curl http://localhost:8080/deduplication/status # Reset deduplication cache curl -X POST http://localhost:8080/deduplication/reset # View deduplication logs docker-compose logs | grep -i duplicate

Problem: Authentication failures
# Check REQUIRE_AUTH setting echo $REQUIRE_AUTH # Validate JWT secret echo $JWT_SECRET | wc -c # Test authentication endpoint curl -X POST http://localhost:8080/auth/login \ -H "Content-Type: application/json" \ -d '{"username": "test", "password": "test"}'
Problem: SSL/TLS certificate issues
# Check certificate files ls -la /etc/ssl/certs/ ls -la /etc/ssl/private/ # Validate certificate openssl x509 -in /etc/ssl/certs/agentic.crt -text -noout # Test SSL connection curl -v https://localhost:8443/health
Problem: Permission denied errors
# Check file permissions ls -la /app/data/ ls -la /app/logs/ # Fix permissions chmod 755 /app/data chmod 644 /app/logs/*.log # Check user permissions id groups

  • Always check logs first when troubleshooting
  • Use docker-compose logs -f to monitor services in real-time
  • Restart individual services with docker-compose restart service-name
  • Check resource usage with docker stats
  • Use docker system prune to clean up disk space
